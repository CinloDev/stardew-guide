## 🇺🇸 English Version

### Header
**Stardew Guide** — Ultimate Interactive Companion for Farmers.

### Metadata
- **Type**: Interactive Game Tool / Tracker
- **Status**: Optimizing / Stable
- **Stack**: Next.js (App Router) · TypeScript · Zustand · TailwindCSS · Prisma

### Description
An advanced, multi-purpose companion application designed to optimize the gaming experience in Stardew Valley. It centralizes seasonal planning, NPC interaction data, and museum collection progress within a premium, mobile-first interface. This project demonstrates a production-ready **feature-based architecture** and high-performance state management using weightless client-side persistence.

### Key Highlights
- **Objectives**: Streamline the tracking of daily tasks and long-term collections without breaking game immersion. Provide a scalable foundation for a future full-stack migration.
- **Strategy**: Implemented a **Game Date Synchronization** system using **Zustand**, allowing users to manually match their in-game time. This state persists across sessions to drive dynamic UI highlights (birthdays, festivals, vendor availability).
- **Results**: Achieved a highly responsive experience with sub-millisecond page transitions and a structured codebase that separates domain logic from UI components, ensuring maintainability and scalability.

## 🇪🇸 Versión en Español

### Encabezado
**Stardew Guide** — El compañero interactivo definitivo para granjeros.

### Metadatos
- **Tipo**: Herramienta Interactiva / Tracker de Juego
- **Estado**: Optimizando / Estable
- **Stack**: Next.js (App Router) · TypeScript · Zustand · TailwindCSS · Prisma

### Descripción
Una aplicación complementaria avanzada y multipropósito diseñada para optimizar la experiencia de juego en Stardew Valley. Centraliza la planificación estacional, los datos de interacción con NPCs y el progreso de la colección del museo en una interfaz premium y optimizada para móviles. Este proyecto demuestra una **arquitectura basada en funcionalidades** (Feature-based) lista para producción y una gestión de estado de alto rendimiento con persistencia ligera en el cliente.

### Puntos Clave
- **Objetivos**: Agilizar el seguimiento de tareas diarias y colecciones a largo plazo sin romper la inmersión del juego. Proporcionar una base escalable para una futura migración a un backend completo.
- **Estrategia**: Implementación de un sistema de **Sincronización de Fecha de Juego** mediante **Zustand**, permitiendo a los usuarios igualar manualmente su tiempo dentro del juego. Este estado persiste entre sesiones para generar alertas dinámicas en la UI (cumpleaños, festivales, disponibilidad de vendedores).
- **Resultados**: Se logró una experiencia altamente fluida con transiciones de página instantáneas y un código estructurado que separa la lógica de dominio de los componentes visuales, asegurando facilidad de mantenimiento y escalabilidad.
